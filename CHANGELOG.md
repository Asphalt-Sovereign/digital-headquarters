# Changelog

All material changes to the Asphalt Sovereign digital headquarters are recorded here.

## 1.1.0 — 2026-07-13

### Added

- Public Evidence Index with seven-part claim chains and explicit Validation in Progress states.
- JSON-driven photography registry for Workshop, Vehicle Testing, Prototype, CAD, PCB, OBD Logging, Team, and Research evidence.
- Seven-stage project lifecycle covering Concept through Production.
- JSON-driven dashboard cards, current sprint, open engineering tasks, research topics, repository counts, log counts, and upcoming milestones.
- Organization-ready repository registry and Open Source page.
- Standards and Safety documentation sections.
- Version numbers and publication states for every technical document.
- Structured weekly log fields for decisions, problems, testing, photographs, commits, reading time, difficulty, and related projects.
- Structured research fields for abstracts, problem statements, technical background, current research, references, future work, related projects, and tags.
- Cross-record JSON validation and deterministic static performance budgets.
- Optional organization-root GitHub Pages mirror workflow.
- Page-level TechArticle, ResearchProject, SoftwareSourceCode, and CollectionPage structured data.

### Changed

- Primary navigation now says Projects instead of Work and exposes Evidence as an engineering destination.
- Mission content now states an engineering method across technology neutrality, sustainable mobility, alternative fuels, intelligence, diagnostics, carbon boundaries, ethics, and validation.
- Company content now explains purpose, method, current competence boundaries, and long-term organizational direction; founder information is limited to accountability.
- Company timeline now follows formation, research, prototype, testing, validation, pilot eligibility, and production-readiness states.
- Project cards and details separate lifecycle, documentation, repository, evidence, and validation status.
- Footer now contains full engineering, company, connection, version, build, and update metadata.
- GitHub publication targets remain non-clickable until verified.
- GitHub Actions now run content, data, lint, type, format, static, SEO, structured-data, and performance checks before deployment.

### Preserved

- Existing visual identity, color system, typography, spacing, component language, dark mode, command palette, search, filters, reading progress, and responsive interaction patterns.
- Every Version 1.0 URL and content slug.
- Next.js App Router, static export, Content Collections, JSON and MDX source architecture, and free GitHub Pages deployment.

### Known public boundaries

- No authentic workshop, vehicle, prototype, CAD, PCB, logging, team, or research photograph is currently published.
- No GitHub organization or repository URL is marked verified.
- No physical integration, alternative-fuel, emissions, durability, pilot, customer, funding, investor, or production claim is made.
- Lighthouse scores require measurement against a deployed build and are not fabricated by the source audit.
